# AgriScheme Data Sources

This document describes the data used in the production-ready market and yield pipeline.

## 1) Yield modeling data (`crop_yield.csv`, `yield_train.csv`, `yield_test.csv`)

### Primary source references
- ICRISAT District-Level Database (India agriculture statistics)
- Directorate of Economics and Statistics (DES), Government of India
- Ministry of Agriculture & Farmers Welfare published crop statistics
- IMD-aligned annual rainfall patterns (used for rainfall trend realism)

### Coverage
- Years: **2001–2022**
- States: **20**
- Crops: **18**
- Seasons: Kharif, Rabi, Summer, Whole Year
- Final rows: **2882** (`crop_yield.csv`)

### Notes on dataset construction
- The dataset is generated by `scripts/generate_yield_dataset.py` using published historical crop and productivity patterns (state/crop/season level).
- It preserves realistic variability by applying:
  - drought-year and flood-year shocks,
  - rainfall–yield sensitivity,
  - irrigation buffering effects,
  - long-run productivity trends.
- Train/test split:
  - `yield_train.csv`: 2305 rows (80%)
  - `yield_test.csv`: 577 rows (20%)

## 2) Market price data (`services/market_service.py`)

### Live source
- **data.gov.in AGMARKNET Mandi Prices API**
- Resource ID: `9ef84268-d588-465a-a308-a864a43d0070`

### Usage mode
- Primary mode: live API fetch + local cache (`data/market_cache.json`)
- Cache TTL controlled by `MARKET_CACHE_TTL` (default 21600 seconds)
- Fallback mode: MSP-based deterministic pricing if API is unavailable or if `MARKET_MODE=msp_only`

## 3) Reproducibility

To regenerate data and retrain model:
1. `python scripts/generate_yield_dataset.py`
2. `python scripts/train_yield_model.py`

Outputs:
- `data/crop_yield.csv`
- `data/yield_train.csv`
- `data/yield_test.csv`
- `models/yield_model.pkl`
- `models/yield_encoders.pkl`
